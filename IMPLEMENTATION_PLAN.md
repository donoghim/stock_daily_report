# 🚀 미국 증시 자산 관리 비서 - 구현 계획서 (Implementation Plan)

## 1. 개요 및 배경
* **발단:** 기존 범용 AI 챗봇(또는 Zapier 연동 봇)을 활용해 매일 아침 미국 증시 시황을 보고받으려 했으나, 잦은 **실행 누락**과 주가/환율 등 핵심 수치에서 치명적인 **할루시네이션(거짓 정보 생성) 오류**가 발생함.
* **목표:** 정확한 데이터 스크래핑과 LLM의 텍스트 요약 능력을 결합한 **Code-driven(코드 기반) 자동화 파이프라인**을 구축하여, 오류 없는 완벽한 일일 시황 리포트를 이메일로 수신함.

---

## 2. 해결 전략: RAG(검색 증강 생성) 기반 아키텍처
LLM이 숫자까지 창작하는 것을 막기 위해, **파이썬 스크립트가 공신력 있는 시장 데이터를 먼저 수집(검색)한 뒤, 이 데이터를 프롬프트에 주입(증강)하여 글짓기만 지시하는 생성 방식**을 채택함.

### 2.1 개발 스택 명세
1. **Language:** `Python 3`
2. **Infrastructure:** `GitHub Actions` (무한 무료 크론 스케줄링 및 가상환경 렌탈)
3. **Market Data:** `yfinance` (지수, 환율, 금리 수집), `pandas_market_calendars` (증시 휴장일 판단)
4. **News Feed:** `feedparser` (야후 파이낸스 실시간 시황 RSS 수집)
5. **AI Engine:** `google-genai` (Google Gemini 1.5 Flash - 비용 효율적이고 강력한 성능)
6. **File Generator:** `markdown`, `pdfkit` (HTML 변환 후 PDF 렌더링)
7. **Email Client:** `smtplib`, `email.mime` (Gmail SMTP 기반 전송)

## 4. 기대 효과
* 파이썬이 주가 데이터를 직접 가져오기 때문에 수치(%) **할루시네이션이 0%로 수렴**됨.
* GitHub Actions 서버 에러가 아닌 이상 약속된 시간에 **실행 누락이 발생하지 않음**.
* (추가 확장성) 향후 개인 관심 종목 코드를 배열에 추가하기만 하면 내 테마에 맞춘 맞춤형 포트폴리오 자산 리포트로 곧장 진화할 수 있음.
